# Delivery Checklist

## Before coding

- Confirm sprint scope.
- Confirm price and payment terms.
- Get repo access and setup instructions.
- Run the app locally.
- Reproduce each issue.
- Write a short implementation plan.

## During coding

- Keep changes scoped.
- Follow existing project conventions.
- Fix one issue at a time.
- Check desktop and mobile layouts.
- Add or update tests when the project supports them.
- Avoid unrelated refactors.

## Before delivery

- Run lint/typecheck/tests/build where available.
- Check the affected pages manually.
- Capture before/after screenshots when useful.
- Prepare a short changelog.
- List any remaining risks or follow-ups.

## Delivery message

- What changed.
- How it was verified.
- Any limitations.
- What the client should review.
