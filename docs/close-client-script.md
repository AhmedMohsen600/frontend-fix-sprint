# Close The Client Script

## Ask About The Issue

Use this:

Can you send me the URL, screenshots or video, expected behavior, current behavior, tech stack, and deadline?

If it is cleanup/refactor work, ask:

- Which exact feature/module should be cleaned?
- What makes it hard to maintain now?
- Are there tests or current behavior I should preserve?
- Should the UI stay exactly the same?
- What files/components are involved?

## Define Scope

For Frontend Fix Sprint:

This fits the Frontend Fix Sprint if we keep it to these items:

- [Fix 1]
- [Fix 2]
- [Fix 3]
- [Fix 4 if needed]
- [Fix 5 if needed]

For Frontend Cleanup Sprint:

This fits the Frontend Cleanup Sprint if we define one exact feature/module:

- Feature/module:
- Current problems:
- Cleanup goals:
- Behavior to preserve:
- Files likely involved:

Anything outside this list can go into a next sprint.

## Say The Price

For fixes:

This would be 250 USD for one Frontend Fix Sprint. That includes 3-5 scoped React/Next.js frontend fixes delivered in 3 days, plus a clean PR or source handoff and verification notes.

For cleanup:

This would be [300/400/500] USD for one Frontend Cleanup Sprint. That includes cleanup of one scoped React/Next.js feature/module in 3-5 days while keeping the same UI and behavior.

## Ask For Payment

Use this:

To reserve the sprint, I take 50% upfront and 50% on delivery before final handoff or merge. Once the first payment is sent and access is ready, I will confirm the final scope and start.

## Avoid Scope Creep

Use this:

That is outside this sprint scope, but I can add it to a next sprint list.

For fixes:

We can either replace one of the current fix items or create a second 250 USD sprint.

For cleanup:

We can either keep this sprint focused on the agreed feature/module or quote a second cleanup sprint for the extra area.

## Upsell Monthly Care

Use this after delivery or when they mention recurring work:

If you expect more frontend fixes or cleanup each month, monthly care is 500 USD/month. It includes two frontend sprints, priority async support, and a small UI/code quality review. We can start with this sprint first, then switch to monthly if it is useful.
