# First Client Execution Pack

## Where To Find The First 30 Leads

- 8 startup founders with live SaaS apps, dashboards, landing pages, or launch posts.
- 6 CTOs/tech leads talking about React, Next.js, frontend debt, hiring, or scaling product teams.
- 6 agencies that ship React/Next.js apps and need overflow support.
- 5 companies hiring frontend contractors.
- 5 older React/Next.js products with visible UI roughness or likely maintenance debt.

## Companies To Target

- Small SaaS teams with live products.
- Agencies with client dashboards, admin panels, or landing pages.
- Teams hiring frontend contractors.
- Startups with old React projects.
- Products where the frontend problem is specific enough to scope.

## Which Offer To Pitch

Pitch Frontend Fix Sprint when they mention:

- UI bugs.
- Responsive problems.
- Dashboard polish.
- Landing page polish.
- Small performance problems.

Pitch Frontend Cleanup Sprint when they mention:

- Messy code.
- Huge components.
- Duplicated logic.
- Bad state management.
- Forms nobody wants to maintain.
- Slow feature changes.
- "We need a frontend refactor."

## Exact First Message

Hey [Name], I am a senior frontend engineer. I help React/Next.js teams with two scoped sprints: visible UI fixes at 250 USD, and messy code cleanup from 300 USD for one feature/module. If you have a page, issue list, or messy module, send it over and I will tell you what fits.

## Follow-Up

Hey [Name], quick follow-up. If the frontend issue or messy React module is still relevant, you can send just the URL, screenshots, or module name. I will keep the scope small and quote it clearly.

## How To Qualify Clients

Ask:

- Is this a visible UI issue or code cleanup task?
- Is the app already built?
- Which page, feature, or module is involved?
- Can the client describe the expected outcome?
- Is there repo or preview access?
- Is the deadline realistic for 3-5 days?
- Can they pay 50% upfront?

Green flags:

- Specific screenshots, issue list, or module name.
- Clear tech stack.
- Existing app URL or repo access.
- They value maintainability or clean delivery.

Red flags:

- Full app rewrite.
- Backend-heavy scope.
- "Refactor everything."
- Unlimited changes.
- Unpaid test task.
- No access but code changes required.

## How To Close A 250 USD Fix Sprint

This fits a Frontend Fix Sprint. I can handle 3-5 scoped frontend fixes in 3 days for 250 USD. I need the app URL, repo access, screenshots or issue list, setup notes, and a test account if needed. Payment is 50% upfront and 50% on delivery.

## How To Close A 300-500 USD Cleanup Sprint

This fits a Frontend Cleanup Sprint if we define one exact feature/module. I will keep the same UI and behavior, split messy components, extract hooks where useful, simplify state/props, remove duplication, and send notes explaining what changed. Price starts at 300 USD and can go up to 500 USD depending on scope.

## How To Upsell To 500 USD/Month

If you expect recurring frontend fixes or cleanup work, monthly care is 500 USD/month. It includes two frontend sprints per month, priority async support, and a small UI/code quality review. We can start with one sprint first, then switch to monthly if it is useful.
